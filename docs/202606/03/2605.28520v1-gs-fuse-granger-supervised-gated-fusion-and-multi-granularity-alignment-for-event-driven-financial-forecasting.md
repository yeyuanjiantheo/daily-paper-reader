## 摘要
准确预测重大金融事件对市场的影响对投资者和政策制定者至关重要。然而，现有的多模态时间序列模型通常对称地融合文本和价格信息，缺乏明确机制判断事件文本何时真正具有预测能力，因而难以利用事件到价格的方向性结构以及文本和价格信号的不同作用。在本研究中，我们提出了GS-Fuse，一个多模态基于事件的预测框架，该框架采用：(i) 一个格兰杰监督的、因果感知的门控融合模块，该模块仅在事件文本相较于历史价格提供增量预测价值时才学习对文本开放；(ii) 一个多粒度对齐机制，同时将高层事件表征和细粒度文本线索与未来市场轨迹对齐。GS-Fuse作为一个灵活、即插即用的适配器，构建在现成的大语言模型和时间序列基础模型之上，可在不同骨干网络和市场环境中实例化。在真实金融数据集上的大量实验表明，GS-Fuse在多种资产和预测周期上均一致优于最先进的时间序列和多模态基线方法。

## Abstract
Accurately forecasting the impact of salient financial events on markets is critical for investors and policymakers. However, existing multimodal time-series models typically fuse text and prices symmetrically, without an explicit way to decide when event text is truly predictive, and thus struggle to exploit the directional event-to-price structure and the heterogeneous roles of textual and price signals. In this work, we propose GS-Fuse, a multimodal event-based forecasting framework that employs (i) a Granger-supervised, causal-aware gated fusion module, which learns to open toward event text only when it provides incremental predictive value beyond historical prices, and (ii) a multi-granularity alignment mechanism that jointly aligns high-level event representations and fine-grained textual cues with future market trajectories. Built as a flexible, plug-and-play adapter on top of off-the-shelf large language models and time-series foundation models, GS-Fuse can be instantiated across diverse backbones and market settings. Extensive experiments on real-world financial datasets show that GS-Fuse consistently outperforms state-of-the-art time-series and multimodal baselines across multiple assets and forecasting horizons.

- **研究问题**：如何构建一个多模态金融预测模型，使之能自适应地、有因果方向性地利用事件文本（如 FOMC 声明、宏观数据发布）与历史价格序列，预测事件后市场的走势。
- **背景与动机**：
  - 现有事件驱动金融预测工作（文本驱动、时间序列驱动、多模态融合）通常对称地融合文本与价格模态，缺乏机制判断“事件文本何时真正具有预测价值”。
  - 对称融合导致模型在事件文本无信息或噪音较大时过度依赖文本，损害预测性能；同时在需要提取细粒度市场冲击线索时，仅依靠实例级对齐会稀释关键信号。
  - 理想方案应体现事件→价格的因果关系，即文本作为外生冲击，仅当其提供超出历史价格增量的预测信息时才被充分采纳。

## 2. 提出的方法论
- **总体思想**：GS-Fuse 是一个即插即用的多模态适配器，包裹在冻结的 LLM 和时间序列基础模型之上，主要包括两个核心创新：
  1. **格兰杰监督的因果门控融合**
  2. **多粒度跨模态对齐**

- **核心机制与流程**：
  - **编码器**：使用冻结的 LLM（如 LLaMA、Phi-3）将事件文本转换为 token 级表征；使用冻结的时间序列基础模型（如 MOMENT、Kronos）将事件前市场窗口转为步级表征；各自经可训练的投影头映射到共享维度 `F=1024`。
  - **多粒度对齐**：
    - 实例级对齐：通过双向交叉注意力交互后池化得到实例级嵌入，并使用 InfoNCE 损失拉近同一事件对的文本和序列表征。
    - 令牌/步级细粒度对齐：计算文本 token 与时间步之间的相似度矩阵，通过软注意力得到每个文本 token 对应的时间序列“软正样本”，并用显著性得分选出 Top-K 锚点 token，以加权 InfoNCE 损失强制文本 token 与相关时间步对齐、同时推开其他样本的时间步。
  - **格兰杰门控融合**：
    - 基于格兰杰因果的思想：比较“仅使用历史价格”的受限预测器（TS-only）与“同时使用价格和事件文本”的非受限预测器（full）的预测损失差 `Δ = ℓ_ts - ℓ_full`。
    - 将损失差通过带温度、截断的 sigmoid 转换为文本有用性软标签 `r`（即文本的“责任目标”）。
    - 门控网络以拼接的实例嵌入为输入，输出特征级的模态权重 `α`，并通过蒸馏损失使平均文本权重 `α_text` 逼近 `r`，从而实现仅在文本能带来增量预测价值时“开门”，否则关闭。
    - 最终融合：`z = α_text * t + α_price * s`。
  - **解码器**：使用轻量 Transformer 解码器（3 层，16 头）将融合向量解码为未来 `H` 步的市场价格序列。
  - **训练方案**：三阶段训练：阶段1仅用时间序列滑动窗口预训练；阶段2仅用文本分支热身；阶段3联合多模态，以预测损失（MSE+MAE）+ 对齐损失 + 门控损失作为总目标。

- **公式/算法简述（文字描述）**：
  - 门控责任目标：`r = σ(clip(Δ/τ_gc, -c, c))`，其中 `σ` 为 sigmoid。
  - 门控损失：`L_gate = MSE(α_text, r)`，`α_text` 为特征门控权重的均值。
  - 总损失：`L_total = L_forecast + λ_align * L_align + λ_gate * L_gate`。

## 6. 主要结论与发现
- GS-Fuse 在所有资产、所有预测周期上一致且显著地优于单向时间序列模型、时间序列基础模型以及已有的多模态基线，尤其在文本噪音较大时优势明显。
- 格兰杰监督门控有效地选择性利用文本：在文本贡献大的资产（如短期国债）上门控开度更高，且全模型优于 TS-only 分支，表明只在文本可带来增量价值时才开门。
- 多粒度对齐（实例级 + token/步级）与门控融合互补，单独移除任何一者均导致性能下降。
- 框架具有骨干网络通用性，但 LLM 与时间序列骨干的匹配会影响性能，合理搭配可进一步获得约 5–10% 的提升。
